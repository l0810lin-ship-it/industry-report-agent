# 第03章：竞争系统与关键控制点

## 要回答的管理问题

竞争不只来自谁，而是谁控制数据、分发、技术、交付与客户关系。

## 输出结构

### 3.1 竞争系统

- 区分直接竞品、替代方案、潜在进入者、互补伙伴和上游依赖。
- 说明各角色在价值链中的位置、目标客户和商业模式。

### 3.2 市场集中度（按模块启用）

当 `market_concentration` 被激活时：

- 给出可比市场份额及口径，计算 CR3 / CR5 / HHI 或说明无法计算的原因。
- 分开判断收入/用户份额与 IP、数据、渠道、供应、资本、监管许可等控制点。
- 只能输出 high / moderate / fragmented / unconfirmed；资源集中不能自动推出“高度垄断”。
- 附支持集中判断的 evidence_id、反例和数据缺口。

### 3.3 关键控制点

| 玩家/角色 | 数据 | 分发/客户关系 | 模型或技术 | 交付能力 | 商业化 | 主要约束 |
|---|---|---|---|---|---|---|

只评价当前题目真正决定胜负的维度。无证据时写“未知”，不得套用固定 AI 功能清单或凭印象打分。

### 3.4 真实案例与可比样本（按模块启用）

当 `real_case_studies` 被激活时，按可见的榜单、收入、采用率、时效、战略相似性或对照结果选择真实案例。Standard 至少 3 个，Deep 至少 5 个。根据行业动态选择拆解维度，不得把虚构概念或内部测试方案当作市场案例。

| 真实案例 | 入选依据 | 行业适配拆解 | 可复用机制 | 不可照搬条件 | evidence_id / URL |
|---|---|---|---|---|---|

### 3.5 护城河与动态变化

- 判断优势是短期功能领先、规模效应、网络效应、数据闭环、切换成本、品牌/信任还是监管许可。
- 说明未来 12–24 个月可能削弱或加强这些优势的变化。

### 3.6 空白与反应路径

- 识别尚未被满足的结构性空白，而不是罗列“竞品没做的功能”。
- 推演目标公司进入后，主要对手可能采取的价格、产品、合作或封锁反应。

### 3.7 本章结论

指出最关键控制点、最危险对手、可合作角色和真正可守住的空白。关键结论附 evidence_id 与 URL。

## 写作要求

- 竞品数量服从研究模式和问题，不为凑表格而扩张。
- 产品公告只能证明“宣称/发布”，不能自动证明采用率或商业效果。
- “头部”“爆款”“行业主流”和“高度集中”都必须有明确样本、分母和时间口径。
