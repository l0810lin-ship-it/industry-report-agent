# 第01章：市场吸引力与进入时点

## 要回答的管理问题

目标市场是否值得进入或加码，为什么是现在？

## 输出结构

### 1.1 市场边界与价值链

- 明确本报告采用的市场定义、地区与时间范围。
- 画出关键价值链角色、利润池和议价关系。
- 区分事实、来源口径与分析推断；不得用未经来源支持的精确市场规模填空。

### 1.2 市场规模：Top-down 与 Bottom-up

- 先明确品类、地区、年份、客户与经济口径（如消费者支出、广告主支出、GMV、净收入或软件支出），不得混用。
- 按 `market-sizing-contract.md` 同时完成 Top-down 与 Bottom-up 测算，展示公式、输入、结果/区间、证据状态与置信度。
- 给出 TAM / SAM / SOM；每个决策关键输入标注 observed、sourced benchmark、assumption 或 unavailable，并附 evidence_id 与 URL。
- 计算两种方法的差异率。差异超过 30% 时不得机械取平均，必须解释市场边界、口径、渗透率、价格、频次或年份差异，并说明哪一种更适合本次决策。
- 给出下行、基准、上行情景。数据不足时保留公式、输入缺口和补证方案，不得跳过或捏造点估计。

### 1.3 吸引力与拐点

- 从需求、供给、技术、监管和资本五个维度判断增长驱动。
- 识别已经发生、正在发生和仍需验证的拐点。
- 给出市场吸引力判断：高/中/低，并列出支持和反对该判断的最强证据。

### 1.4 行业趋势与区域进入顺序（按模块启用）

当 `trend_inference` 或 `geographic_sequencing` 被激活时：

- 不等待用户给出路线；重建至少 3 个可比主体的实际进入时间线，列出主体、地区、首次进入时间、经营方式、前置能力、结果和 evidence_id。
- 显示符合、不符合及未知的主体数量，以 `n/N` 报告样本结果，区分 supported_in_sample / emerging_signal / mixed / inconclusive；同一时期覆盖多个地区不能证明先后顺序。
- `supported_in_sample` 只能表述为“在所定义样本中观察到重复路径”。除非证明样本具有代表性，否则不得写成确定的全行业路径；小样本只能称为粗略信号。
- 比较至少两条候选路径及每阶段的市场价值、获客/变现、文化改造、合规/IP、竞争、渠道和资本风险。
- 明确回答为什么不直接进入价值最高的市场；允许结论是“直接进入更优”。
- 分开写“所定义样本中观察到的路径信号”和“目标公司应该采用的路径”。不得因为样本中出现重复路径就自动推荐照搬。

### 1.5 进入时点与情景

给出基准、上行、下行情景，并说明：

| 情景 | 触发条件 | 业务影响 | 可观测信号 | 管理动作 |
|---|---|---|---|---|

### 1.6 本章结论

用不超过 5 条结论回答：是否值得、为何现在、最大不确定性是什么。每条关键结论附 evidence_id 与 URL。

## 写作要求

- 不强制近三年趋势表；没有同口径数据就不做伪趋势。
- 不把“大市场”自动等同于“适合目标公司”。
- 跨境课题不得只做静态地区横向表，必须检查行业主体的真实进入时间线、重复路径和反例。
- Standard/Deep 的正式报告缺少任一市场规模方法、计算过程或差异解释，都视为未完成。
- 结论必须能被后续战略选择直接使用。
