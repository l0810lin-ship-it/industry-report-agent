# 管理层摘要

基于五章内容生成一页内的管理层摘要，不能引入正文中不存在的新事实。

严格包含：

1. **一句话结论**：是否进入/加码，以及推荐的具体赛道、首发产品和产品入口。禁止只写“平台、生态、编排层、能力层、解决方案”等抽象定位。
2. **首发下注卡**：用紧凑表格列出目标赛道、首发品类/产品楔子、目标用户、核心任务、产品入口、产品形态、供给方式、交易/履约边界、商业模式、首发区域、主要竞品、独特优势、90天证伪指标和明确不做。证据不足的字段写 `TBD—证据不足`，并将建议降级为条件性推荐。
3. **为什么是现在**：最多 3 条，包含市场或竞争拐点。
4. **为什么我们能赢**：最多 3 条，区分已验证能力与待验证假设。
5. **关键数字与证据**：只保留会改变决策的 3–5 项，附 evidence_id 与 URL。
6. **关键趋势与前提状态**：说明候选趋势是 supported_in_sample / emerging_signal / mixed / inconclusive，必须同时给出 `n/N`、样本边界和外推限制，并列出被支持、部分支持或推翻的关键前提；未启用相关模块时省略。
7. **最大风险/反方观点**：至少 2 条，不回避与推荐冲突的证据。
8. **未来 90 天**：3 项动作和对应决策门。
9. **需要管理层批准**：列出明确决策，不写泛泛建议。

使用短句和表格，优先保证下注卡完整；若与 800 中文字限制冲突，可适度超出。若证据不足以支持正式推荐，明确输出“条件性推荐”并列出补证条件。
